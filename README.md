# **1️⃣ プロジェクトの目的**

`英単語動画（InsertOneWord）の視聴者が、`  
`動画で見た単語を短時間で復習できる場所を提供する。`

`このプロジェクトの目的は`  
`「収益化」ではなく、`  
`動画 → アプリ → 再訪が起きるかを検証すること。`

---

# **2️⃣ 背景**

* 英単語動画は継続中  
   TODO  
* 受け皿（学習場所）が未整備  
* 教育系に収益軸を集中する方針  
* STEP1（集客）は進行中  
* STEP2（受け皿検証）に入る段階

---

# **3️⃣ プロジェクトのゴール**

### **🎯 成功条件**

`動画視聴者の一部が`  
`翌日以降もアプリを開く。`

### **❌ ゴールではないもの**

* 売上  
* DL数  
* ストア評価  
* デザイン完成度

---

# **4️⃣ MVP仕様（確定版）**

## **機能**

1. 英単語 \+ 日本語表示  
2. 「覚えた」「まだ」ボタン  
3. 「まだ」を優先したランダム出題  
4. 単語ごとの動画リンク  
5. 再訪判定（localStorage）

## **実装方針**

* 完全静的構成  
* サーバーDBなし  
* 外部送信なし（フェーズ1）  
* 無料ホスティング

---

# **5️⃣ 技術方針**

## **フェーズ1**

* Webアプリ（スマホ特化）  
* PWA対応  
* GitHub Pages or Vercel  
* localStorage保存

月額固定費：ほぼ0円

---

## **フェーズ2（回ったら）**

* 再訪ログ送信  
* 分析導入  
* ネイティブ化検討

---

# **6️⃣ やらないこと**

* 収益化設計  
* ユーザー管理  
* 課金機能  
* レベル分け  
* テスト機能  
* 同期機能

---

# **7️⃣ リスク管理**

| リスク | 対応 |
| ----- | ----- |
| 静的で足りない | 成功時のみ動的化 |
| 運用費増大 | 無料ホスティング |
| 法的リスク | 外部送信なし |

---

# **8️⃣ 判断ポイント（撤退ライン）**

以下の場合は停止または再設計：

* 再訪がほぼゼロ  
* 動画からの遷移が極端に少ない  
* 開発負担が本業に影響

---

# **9️⃣ このプロジェクトの位置づけ**

`別収入全体プロジェクトの`  
`「教育軸の成立検証フェーズ」`

成功すれば：

→ 教育系プロダクト化  
→ サブスク化  
→ ネイティブ展開
→ React化
→ AWS化

失敗しても：

→ 固定費ほぼゼロ  
→ 学習データ獲得
